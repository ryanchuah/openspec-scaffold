## Prior change — verify-multimodel-gate SHIPPED (2026-06-16)

Independent multi-model verification passes added as hard gates in the verify step: a new delegated `openspec-verifier` agent (read-only on files, bash-capable, `edit: deny`) runs the same behavioral review the self-review runs — read diffs, full test suite, real output — and emits a machine-discriminable verdict (`## Verify Pass` / `VERDICT: READY` or `NEEDS REVISION`). The pass chain depends on platform: a Claude Code orchestrator runs self-review → `deepseek-v4-pro` → `deepseek-v4-flash` (three independent passes); an OpenCode orchestrator runs self → flash only (already deepseek-pro, so only flash adds model diversity). Each pass is a hard gate; on NEEDS REVISION the orchestrator fixes via re-delegation and reruns the failed pass and every pass after it (never earlier ones), escalating to the operator after three cycles. The verifier's hardening extends the existing noninteractive-delegation-safety posture: the Claude Code `opencode run` verifier invocations close stdin, the OpenCode Task-tool path is exempt, and the verifier takes executor-style `external_directory` containment (a third permission category — bash-capable but write-denied — between reviewer and executor). The orchestrator judges every finding from disk, not on the verifier's word. Verify verdict: READY FOR ARCHIVE — behavioral review passed; all three passes (self/pro/flash) returned READY with no defects; the V6 live probe confirmed both verifier tiers loaded the real agent and emitted the machine-discriminable verdict block.
