## 2. Claims-ledger staleness detector (flash apply-executor)

- [x] 2.1 Add `_check_claims_ledger_staleness(root: Path) -> list[Finding]` to `scripts/knowledge_lint.py`, mirroring the guard idiom of `_check_post_close_ledger` (glob → read → marker-gate → per-entry check → append). Algorithm: `for ledger in sorted(root.glob("knowledge/reference/*.md"))`: read text (skip on OSError); `if "format: product-audit/v1" not in text: continue`. Scan lines with an `in_manifest` flag set True when a stripped line equals `## Covered promise-surface files` and reset False on any other line whose stripped form starts with `## `. For each line while `in_manifest`, parse with `re.match(r"\s*-\s*(?P<path>.+?)\s*[—–-]+\s*sha256:(?P<hash>[0-9a-fA-F]{64})\b", line)`; **no match → skip the line silently** (lenient parse — malformed/vacuous never flags; blank lines within the manifest section won't match and are silently skipped). For a match: resolve `path` relative to `root`; if it is not an existing file → append `Finding("claims-ledger-staleness", <ledger relpath>, <lineno>, "claims ledger covers a missing promise-surface file: <path>")`; else compute `hashlib.sha256(target.read_bytes()).hexdigest()` and, if it differs (case-insensitive) from the recorded hash → append `Finding("claims-ledger-staleness", <ledger relpath>, <lineno>, "claims ledger stale: <path> content changed since last reconciliation")`. Use the module's existing `_relpath` helper for paths. Detect-only; no writes. Ensure `import hashlib` is present at module top (add if missing).
