## 1. Build the linter `scripts/status_lint.py` (scaffold)

- [x] 1.1 Create `/home/pang/Projects/openspec-scaffold/scripts/status_lint.py` — a **stdlib-only** Python 3.12 script (no third-party imports), mirroring the structure/conventions of `scripts/scaffold_check.py` (module docstring; a `main(argv=None) -> int` entry point; `if __name__ == "__main__": sys.exit(main())`). It enforces, for a given repo, the mechanical invariants of `AGENTS.md §"State, write discipline, and the archive-as-handoff rule"` (with the small pragmatic exempt-heading extension documented in `notes.md`). CLI: one optional positional `repo_root` (default = the repo containing the script, i.e. `Path(__file__).resolve().parent.parent`). Exit code **0** = all clean, **2** = one or more hard violations (matches `scaffold_check.py` / the Claude-Code PreToolUse convention).

- [x] 1.2 **STATUS.md checks** — operate on `<repo_root>/STATUS.md`; if the file is absent, skip with no error and contribute no violations. Parse into sections split on lines matching `^## ` (text before the first `## ` is the *preamble*). For each `## ` section compute a *normalized heading* = heading text with the leading `## ` removed, lowercased, whitespace-collapsed. A section is **exempt (structural)** if its normalized heading **exactly equals** an entry in a module constant `EXEMPT_HEADINGS = {"current state", "immediate next action", "done", "pointers"}` (exact match, **not** prefix — a change-entry whose title merely begins with one of these words, e.g. `## Done-archive migration`, must still count); the preamble is always exempt. (AGENTS.md's prose names only the preamble + `## Immediate next action`; `current state`/`done`/`pointers` are a pragmatic extension for the structural sections real STATUS.md files use — see `notes.md`.) Every non-exempt `## ` section is a **change-entry** (matches the rule: "Any `##` section that narrates a shipped change counts … regardless of heading title", e.g. `## Operations …`).
  - **C1 — cap count:** if there are more than **3** change-entry sections, emit one violation naming the count and listing the change-entry headings beyond the 3 most recent (STATUS.md is newest-first, so the excess are the *last* change-entry sections in file order).
  - **C2 — per-entry word budget:** for each change-entry section, compute its word count = number of `\S+` tokens in the section **body** (all lines after the heading line, up to the next `## ` or EOF) **after removing fenced code blocks** (the lines from an opening ```` ``` ```` fence through its closing fence). If the count exceeds **150**, emit a violation naming the heading and its measured word count.

- [x] 1.3 **decisions.md checks** — operate on `<repo_root>/ai-docs/decisions.md`; skip with no error if absent. Add a CLI flag `--since YYYY-MM-DD` defaulting to **`2026-06-18`** (the adoption date of the decisions-entry rule; document this in the docstring). Split into entries on `^## ` headings. For every entry, first remove fenced code blocks from its body (same as 1.2) so metadata/word checks ignore code examples, then parse its `**Date:** YYYY-MM-DD` value. **Enforcement applies only to entries whose Date parses as a real date on/after `--since`** — this honors "no retroactive backfill": legacy entries predate the `**Status:**` requirement (they carry a `**Date:**` but no Status), and the placeholder template `**Date:** YYYY-MM-DD` does not parse, so all are skipped. For each in-scope entry:
  - emit a violation if the entry body does **not** also contain a `**Status:**` line;
  - if the entry's normalized heading starts with a change-record token (regex `^(fix|add|tune)-`, anchored at the start of the normalized heading — exactly the `fix-*`/`add-*`/`tune-*` entry-name prefixes named in AGENTS.md, so a prose heading like `## The add-subtract module` does not match), compute the body word count by the same `\S+` / code-fence rule as 1.2 and emit a violation if it exceeds **300**.
  Entries with no `**Date:**` line, an unparseable Date, or a Date before `--since` are skipped (legacy/template); do not enforce Status or the word budget on them.

- [x] 1.4 **Output format:** first line `status_lint: checking <repo_root>`. For each file checked print `<relpath>: OK` or `<relpath>: FAIL` followed by indented (`  `) violation lines (terse and specific — heading + measured number, in the style of `scaffold_check.py`). End with `status_lint: OK` (return 0) or `status_lint: FAILED — <n> violation(s)` (return 2).

  - [x] **Graceful absence:** a repo with neither `STATUS.md` nor `ai-docs/decisions.md` → `main` returns 0; and the asymmetric case (STATUS.md present, `ai-docs/decisions.md` absent) → the decisions checks are skipped without error.
